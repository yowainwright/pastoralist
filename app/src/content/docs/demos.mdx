## Demo Descriptions

### Basic Override Management
**Path:** `/demo/basic`

Perfect for beginners! Learn how pastoralist:
- Tracks package overrides
- Creates documentation automatically
- Shows dependency relationships
- Cleans up unused overrides

**Try it:** See how overrides are documented and managed automatically.

---

### Security Vulnerability Detection
**Path:** `/demo/security`

Explore advanced security features:
- Scan for vulnerable dependencies
- Auto-fix with one command
- Interactive mode for selective fixes
- Backup and rollback capabilities

**Try it:** Fix real vulnerabilities in demo packages!

---

### Monorepo Workspaces
**Path:** `/demo/workspaces`

See monorepo management in action:
- Root-level override control
- Cross-package dependency tracking
- Unified version management
- Workspace-aware cleanup

**Try it:** Manage overrides across multiple packages.

---

### Migration from Manual Overrides
**Path:** `/demo/migration`

Learn to migrate existing projects:
- Analyze current overrides
- Remove unnecessary ones
- Add documentation
- Set up automation

**Try it:** Transform messy overrides into documented, maintained ones.

## Running Demos Locally

While StackBlitz is the easiest way, you can also run demos locally:

```bash
# Clone the repository
git clone https://github.com/yowainwright/pastoralist.git

# Navigate to a demo
cd pastoralist/demo/basic

# Install and run
bun install
bun start
```

## Demo Features

Each demo includes:
- **Real package.json files** - See actual configurations
- **Interactive scripts** - Run commands and see results
- **Before/after comparisons** - Understand the transformation
- **Full documentation** - Learn while doing

## Creating Custom Demos

Want to create your own demo? It's easy!

1. Fork any demo as a starting point
2. Modify for your use case
3. Push to GitHub
4. Open via StackBlitz URL:
   ```
   https://stackblitz.com/github/[username]/[repo]/tree/[branch]/[path]
   ```

## Tips for Using Demos

- **Edit files freely** - Changes are sandboxed
- **Run commands** - Full terminal available
- **Fork to save** - Create your own version
- **Share URLs** - Each demo has a unique URL
- **No installation** - Everything runs in browser
